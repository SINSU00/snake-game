## 3. 게임 기획

### 3.1 핵심 메카닉
- **뱀 이동**: 20x20 그리드 위에서 일정 간격으로 한 칸씩 이동
- **먹이 섭취**: 먹이를 먹으면 뱀 길이 +1, 점수 획득
- **게임 오버 조건**: 벽 충돌(벽 통과 모드 OFF 시) 또는 자기 몸 충돌
- **속도 증가**: 점수가 올라갈수록 이동 속도 점진적 증가

### 3.2 난이도 시스템
게임 시작 전 오버레이 화면에서 3단계 난이도를 선택할 수 있다. 게임 오버 후에도 난이도를 변경하여 재시작할 수 있다.

| 항목 | 쉬움 | 보통 (기본) | 어려움 |
|------|------|------------|--------|
| 시작 이동 간격 | 160ms (느림) | 125ms | 85ms (빠름) |
| 최대 속도 제한 | 90ms | 62.5ms | 40ms |
| 속도 증가율 (점수당) | 0.3 (완만) | 0.5 | 0.8 (급격) |
| 콤보 인정 시간 | 3초 (넉넉) | 2초 | 1.5초 (빡빡) |
| 버튼 테마 색상 | 파란색 (`#41c8ff`) | 초록색 (`#00ff41`) | 빨간색 (`#ff4141`) |

> 속도 공식: `moveInterval = max(minInterval, startInterval - score * speedIncrease)`

### 3.3 콤보 시스템
- 난이도별 콤보 인정 시간 내에 연속 섭취 시 콤보 배율 증가 (최대 x5)
- 콤보 배율에 따라 획득 점수 배수 적용 (기본 10점 x 콤보)
- 콤보 인정 시간 초과 시 콤보 배율 리셋

### 3.4 점수 체계
| 항목 | 값 |
|------|-----|
| 기본 먹이 점수 | 10점 |
| 콤보 최대 배율 | x5 (최대 50점/먹이) |
| 황금 먹이 점수 | 50점 x 콤보 배율 |
| 최고 점수 | localStorage에 영구 저장 |

### 3.5 난이도별 속도 곡선

**쉬움** (시작 160ms, 증가율 0.3)
| 점수 | 이동 간격(ms) | 초당 이동 횟수 |
|------|--------------|---------------|
| 0 | 160ms | ~6회 |
| 100 | 130ms | ~8회 |
| 200 | 100ms | 10회 |
| 233+ | 90ms (최대) | ~11회 |

**보통** (시작 125ms, 증가율 0.5)
| 점수 | 이동 간격(ms) | 초당 이동 횟수 |
|------|--------------|---------------|
| 0 | 125ms | 8회 |
| 50 | 100ms | 10회 |
| 100 | 75ms | ~13회 |
| 125+ | 62.5ms (최대) | 16회 |

**어려움** (시작 85ms, 증가율 0.8)
| 점수 | 이동 간격(ms) | 초당 이동 횟수 |
|------|--------------|---------------|
| 0 | 85ms | ~12회 |
| 30 | 61ms | ~16회 |
| 56+ | 40ms (최대) | 25회 |

### 3.6 벽 통과 모드
게임 시작 전 오버레이에서 벽 통과 모드를 켜기/끄기 할 수 있다.

| 항목 | 끄기 (기본) | 켜기 |
|------|------------|------|
| 벽 충돌 | 게임 오버 | 반대편에서 나옴 |
| 자기 몸 충돌 | 게임 오버 | 게임 오버 |
| 좌표 처리 | 범위 초과 시 종료 | 모듈로 연산으로 좌표 감싸기 |

- 켜기 시: 벽에 부딪히면 반대편에서 나옴 (좌→우, 상→하)
- 끄기 시: 기존과 동일하게 벽 충돌로 게임 오버
- 두 모드 모두 자기 몸 충돌은 게임 오버

### 3.7 특수 먹이 시스템
일반 먹이를 먹을 때 확률적으로 특수 먹이가 등장한다. 제한 시간 내에 먹어야 하며, 사라지기 2초 전부터 빠르게 깜빡인다.

#### 3.7.1 황금 먹이
| 항목 | 값 |
|------|-----|
| 등장 확률 | 15% (일반 먹이 섭취 시) |
| 기본 점수 | 50점 x 콤보 배율 |
| 유지 시간 | 5초 |
| 외형 | 금색(`#ffd700`) 별 모양 |
| 사운드 | 600Hz→1200Hz 상승음 (0.15초) |

#### 3.7.2 느림 먹이
| 항목 | 값 |
|------|-----|
| 등장 확률 | 10% (일반 먹이 섭취 시) |
| 효과 | 이동 속도 50% 감소 (5초간) |
| 유지 시간 | 7초 |
| 외형 | 시안색(`#00ffff`) 마름모 모양 |
| 사운드 | 600Hz→1200Hz 상승음 (0.15초) |

### 3.8 뱀 스킨 커스터마이징
게임 시작 전 오버레이에서 뱀의 색상을 선택할 수 있다. 선택한 색상은 뱀 몸체, 발광 효과, 격자 색상에 모두 적용된다.

| 스킨 | 색상 코드 | RGB |
|------|----------|-----|
| 초록 (기본) | `#00ff41` | 0, 255, 65 |
| 블루 | `#41c8ff` | 65, 200, 255 |
| 핑크 | `#ff69b4` | 255, 105, 180 |
| 골드 | `#ffd700` | 255, 215, 0 |

## 4. 조작 방식

### 4.1 키보드 (데스크톱)
| 키 | 동작 |
|----|------|
| Arrow Keys / WASD | 방향 전환 |
| Space | 게임 시작 / 재시작 |
| P / ESC | 일시정지 / 재개 |

### 4.2 터치 (모바일)
| 제스처 | 동작 |
|--------|------|
| 스와이프 (30px 이상) | 방향 전환 |
| 탭 (게임 오버 시) | 재시작 |
| 탭 (일시정지 중) | 게임 재개 |

### 4.3 역방향 방지
- 현재 진행 방향의 반대 방향 입력은 무시됨
- 예: 오른쪽 이동 중 왼쪽 입력 -> 무시

### 4.4 일시정지
- `P` 키 또는 `ESC` 키로 게임 일시정지/재개 토글
- 일시정지 중에는 뱀 이동 및 방향 입력이 모두 정지
- 일시정지 오버레이에 "일시정지" 텍스트와 안내 표시
- 재개 시 타이밍 보정 (일시정지 시간만큼 lastMoveTime 보정)
- 게임이 실행 중이 아닐 때(시작 전/게임오버)는 무시

## 5. 비주얼 디자인

### 5.1 테마
- **네온 사이버펑크** 스타일
- 어두운 배경 (`#0f0f1e` ~ `#1a1a2e` 그라디언트)
- 뱀 색상: 스킨에 따라 변경 (기본 네온 그린 `#00ff41`)
- 네온 레드/핑크 (`#ff0040`) 일반 먹이 색상
- 금색 (`#ffd700`) 황금 먹이 색상
- 시안 (`#00ffff`) 느림 먹이 색상

### 5.2 시각 효과
| 효과 | 구현 방식 |
|------|----------|
| 타이틀 글로우 | CSS `text-shadow` + `@keyframes glow` 애니메이션 |
| 뱀 발광 | Canvas `shadowBlur` + `shadowColor` (스킨 색상 적용) |
| 뱀 그라데이션 | 머리에서 꼬리로 투명도 감소 (`1.0` -> `0.5`) |
| 먹이 펄스 | `Math.sin(Date.now())` 기반 크기/밝기 진동 |
| 먹이 폭발 | 파티클 시스템 (15개 파티클, 중력 + 페이드아웃) |
| 특수 먹이 폭발 | 파티클 시스템 (20개 파티클, 먹이 색상 적용) |
| 특수 먹이 긴급 깜빡임 | 사라지기 2초 전부터 빠른 깜빡임 |
| 콤보 텍스트 | CSS 애니메이션 (떠오르며 사라짐) |
| 일시정지 텍스트 | 노란색 + 크기 펄스 애니메이션 |
| 오버레이 블러 | CSS `backdrop-filter: blur(5px)` |

### 5.3 뱀 디자인
- 각 세그먼트: 둥근 사각형 (2px 내부 패딩)
- 머리에 눈 렌더링 (이동 방향에 따라 위치 변경)
- 머리가 가장 밝고 꼬리로 갈수록 어두워짐
- 선택한 스킨 색상이 뱀 전체에 적용

### 5.4 특수 먹이 디자인
| 먹이 | 모양 | 색상 |
|------|------|------|
| 황금 먹이 | 별 (5꼭짓점) | 금색 (`#ffd700`) |
| 느림 먹이 | 마름모 (다이아몬드) | 시안 (`#00ffff`) |

## 6. 사운드 시스템

Web Audio API (`AudioContext`)를 사용한 프로시저럴 사운드 생성. 외부 오디오 파일 없음.

### 6.1 먹이 섭취 사운드
- 오실레이터: 400Hz -> 800Hz (0.1초간 상승)
- 볼륨: 0.3 -> 0.01 (0.1초간 감쇠)

### 6.2 게임 오버 사운드
- 오실레이터: 300Hz -> 100Hz (0.5초간 하강)
- 볼륨: 0.3 -> 0.01 (0.5초간 감쇠)

### 6.3 특수 먹이 섭취 사운드
- 오실레이터: 600Hz -> 1200Hz (0.15초간 상승)
- 볼륨: 0.3 -> 0.01 (0.15초간 감쇠)
- 일반 먹이보다 높은 음역대로 차별화

## 7. 기술 구현 상세

### 7.1 게임 루프
```
requestAnimationFrame 기반 루프
├── 캔버스 클리어
├── 그리드 렌더링 (단일 Path로 최적화, 스킨 색상 적용)
├── 일반 먹이 렌더링 (펄스 효과)
├── 특수 먹이 렌더링 (별/마름모 + 긴급 깜빡임)
├── 뱀 렌더링 (글로우 + 눈, 스킨 색상 적용)
├── 파티클 업데이트/렌더링
└── 이동 타이밍 체크 -> moveSnake() 호출 (일시정지 시 건너뜀)
```

- 렌더링: 매 프레임 (`~60fps`)
- 게임 로직(이동): `moveInterval` 간격으로 분리
- 일시정지 시: 렌더링은 계속, 로직(이동)만 정지

### 7.2 충돌 감지
- **벽 충돌 (벽 통과 OFF)**: 좌표가 0 미만 또는 GRID_SIZE 이상인지 체크 → 게임 오버
- **벽 통과 (벽 통과 ON)**: 좌표가 범위를 벗어나면 반대편으로 감싸기 (모듈로 연산)
- **자기 충돌**: `Array.some()`으로 머리 좌표와 몸체 좌표 비교 → 항상 게임 오버

### 7.3 먹이 생성
- 랜덤 좌표 생성 후 뱀 몸체 및 특수 먹이와 겹치지 않는지 검증
- 겹치면 재생성 (while 루프)

### 7.4 특수 먹이 생성
- 일반 먹이 섭취 시 확률 판정 (느림 10%, 황금 15%)
- 이미 특수 먹이가 있으면 생성하지 않음
- 뱀/일반 먹이와 겹치지 않는 위치에 배치
- 제한 시간 후 setTimeout으로 자동 제거

### 7.5 파티클 시스템
- `Particle` 클래스: 위치, 속도, 생명력, 색상 (커스텀 색상 지원)
- 매 프레임 물리 업데이트 (속도 + 중력)
- 생명력 0 이하 시 배열에서 제거 (역순 순회 + splice)
- 특수 먹이 파티클은 해당 먹이 색상 사용

### 7.6 반응형 캔버스 크기
```javascript
CELL_SIZE = min(
    floor((화면 너비 - 100) / 20),
    floor((화면 높이 - 300) / 20)
)
```
- 화면 크기에 맞춰 셀 크기 자동 계산
- 모바일 미디어 쿼리 (`max-width: 768px`)로 UI 요소 축소

### 7.7 데이터 저장
- `localStorage.snakeHighScore`에 최고 점수 영구 저장
- 페이지 로드 시 자동 복원

### 7.8 일시정지 시스템
- `gamePaused` 플래그로 상태 관리
- 일시정지 시 `pausedTimestamp` 기록
- 재개 시 `lastMoveTime += (현재시간 - pausedTimestamp)`로 시간 보정
- 일시정지 중 방향키 입력 무시
- 모바일: 터치로 재개 가능

### 7.9 속도 감소 효과
- `baseMoveInterval`: 속도 감소 전 원래 이동 간격
- `moveInterval`: 실제 적용되는 이동 간격
- 느림 먹이 섭취 시 `moveInterval = baseMoveInterval / 0.5` (2배 느려짐)
- 5초 후 `moveInterval = baseMoveInterval`로 복귀
- 효과 중 새 느림 먹이 먹으면 타이머 리셋 (중복 방지)

## 10. 성능 최적화 포인트

1. **그리드 렌더링**: 개별 선 대신 단일 `beginPath()` + `stroke()`로 배치 드로잉
2. **파티클 제거**: 역순 순회로 splice 시 인덱스 꼬임 방지
3. **렌더링/로직 분리**: 렌더링은 60fps, 게임 로직은 별도 타이밍으로 분리
4. **역방향 입력 차단**: 불필요한 상태 변경 방지
5. **프로시저럴 오디오**: 외부 파일 로딩 없이 즉시 사운드 생성
6. **특수 먹이 단일 제한**: 동시에 하나만 존재하여 메모리/렌더링 부담 최소화
